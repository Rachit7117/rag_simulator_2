How this Simulator Works

This simulator demonstrates the complete RAG architecture:


1. Your knowledge base gets divided into smaller chunks.

2. Each chunk is converted into vector embeddings (numerical representation of meaning).

3. These vectors are stored in a vector database.

4. The user query is also converted into a vector embedding.

5. Similarity search compares the query vector with chunk vectors.

6. Most relevant chunks are retrieved and passed into the LLM prompt.

7. The LLM generates a grounded response instead of hallucinating.
